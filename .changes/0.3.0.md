## 0.3.0 - 2026-06-26
### Added
* LlmJudge: an LLM grades the AI's output against criteria you write and returns a pass/fail verdict, a 0-1 score, and a reason. Configure it with criteria, scoring steps, few-shot examples, and a rubric.
* Llm: a single seam for calling language models (litellm under the hood), shared by solvers and the LLM judge, so you can plug in your own model client.
* observed_equivalence(), judged_equivalence(model), and sql_equivalence_judge(model): ready-made query-equivalence scorers that confirm by comparing query structure first, then fall back to running both queries or asking an LLM judge.
* Scorers now return a three-valued verdict — 'pass', 'fail', or 'inconclusive' — alongside an optional 0-1 score, so a scorer can report when it cannot decide rather than being forced to pass or fail.
* basis on every result records how strong the evidence is: 'proven' (sound on all data), 'observed' (seen on the data that was run), or 'judged' (an LLM's judgment).
### Removed
* query_equivalence(): replaced by observed_equivalence(), which runs the same compare-structure-then-run check and can also report 'inconclusive'.
